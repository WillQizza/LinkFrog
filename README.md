# LinkFrog

LinkFrog is a link shortening service that allows you to customize what link codes you want, and also whitelist it so only specific users authenticated through Google OAuth can shorten links.

## Features
- Google OAuth
- Whitelisted users
- Custom/autogenerated shortened links
- Some snazzy frog themed colors

## Tech Stack
**Backend:** Go, MySQL, Chi
**Frontend:** TypeScript, Nginx, React
**Infrastructure:** Docker

## How to Run

```bash
# 1. Copy and configure environment variables
cp .env.example .env

# Fill in your Google OAuth credentials and change the passwords/secrets

# 2. Build and run
docker-compose up -d --build
```

The app will be available at `http://localhost:3000`.
The first user to sign in via Google is automatically whitelisted and can invite others from the dashboard.

## Environment Variables

| Variable               | Description                                                                |
| ---------------------- |----------------------------------------------------------------------------|
| `DB_DSN`               | MySQL connection string                                                    |
| `MYSQL_DATABASE`       | Database name                                                              |
| `MYSQL_USER`           | Database user                                                              |
| `MYSQL_PASSWORD`       | Database password                                                          |
| `MYSQL_ROOT_PASSWORD`  | MySQL root password                                                        |
| `GOOGLE_CLIENT_ID`     | Google OAuth client ID                                                     |
| `GOOGLE_CLIENT_SECRET` | Google OAuth client secret                                                 |
| `GOOGLE_REDIRECT_URL`  | OAuth callback URL (e.g. `http://localhost:3000/api/auth/google/callback`) |
| `AUTH_REDIRECT_URL`    | Frontend redirect after login (e.g. `http://localhost:3000`)               |
| `JWT_SECRET`           | Secret key for signing JWT tokens                                          |

See [.env.example](.env.example) for defaults.
